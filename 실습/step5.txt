현재 디렉토리의 개발 티켓, AORR.md, MEMORY.md를 읽고 Self-Correcting TDD 루프를 정확히 1회만 실행해줘.

실행 순서:

1. MEMORY.md의 목표, 범위, 금지사항, 현재 상태를 확인
2. 이번 루프에서 해결할 가장 작은 작업 하나를 선택
3. 변경 전 Verifier를 실행해 현재 상태를 기록
4. 실패 원인을 CODE, TEST, SPEC, ENVIRONMENT, PERMISSION, UNKNOWN 중 하나로 분류
5. 하나의 원인에 대해서만 최소 수정 수행
6. 동일한 Verifier를 다시 실행
7. 실행 결과를 MEMORY.md의 Execution Log에 추가
8. 현재 상태와 다음 행동을 MEMORY.md에 업데이트
9. 정확히 한 번의 Act-Observe-Reason 사이클 후 중지

가능하면 Claude Code CLI를 독립 Verifier로 사용해.
Sonnet 5가 실제로 사용 가능하면 사용하고, 그렇지 않으면 사용 가능한 Sonnet 모델을 사용한 뒤 모델명을 기록해.

다음 작업은 하지 마:
- GitHub Pages 배포
- 원격 저장소 push
- 테스트 삭제 또는 완화
- API 스펙 변경
- 인증·권한 변경
- 대규모 리팩토링
- 비밀키나 토큰 출력

최종적으로 다음을 보고해:
- 수행한 Act
- 변경한 파일
- 실행한 Verifier
- 테스트 결과
- 현재 상태: PASSED, FAILED, BLOCKED, HITL_REQUIRED
- 다음 루프에서 수행할 작업

실패하거나 불확실한 내용은 [사람 확인 필요]로 표시해줘.
