CHANGE_REQUEST.md, AORR.md, MEMORY.md를 읽고 사용자 변경 요청에 대한 새로운 Self-Correcting TDD 루프를 시작해줘.

기존 완료 기록과 테스트 결과는 보존하고, 이번 변경 요청에 필요한 범위만 수정해.

반드시 다음을 검증해:
1. 새로운 변경 요청의 완료 기준
2. 기존에 통과했던 기능의 회귀 테스트
3. 모바일 및 데스크톱 화면
4. Build와 GitHub Pages 호환성
5. 콘솔 오류와 깨진 링크 여부

한 번의 Retry에서는 하나의 원인만 수정하고 최대 3회까지 재시도해.
동일 오류가 2회 반복되면 중지해.

모든 테스트가 통과해도 바로 배포하지 말고 상태를
USER_REVIEW_REQUIRED로 설정해.

다음 내용을 사용자에게 보고한 뒤 중지해:
- 사용자 요청 중 반영된 항목
- 변경 파일
- 테스트 결과
- 직접 확인해야 할 화면과 동작
- 배포 전 사람 확인 필요 항목

사용자가 결과를 승인한 이후에만 commit, push, GitHub Pages 배포를 수행해.
승인하면, github_token.txt의 token=""을 사용해서 [본인].github.io 에 배포해
