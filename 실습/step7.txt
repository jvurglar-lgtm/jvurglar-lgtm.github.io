현재 디렉토리의 개발 티켓, AORR.md, MEMORY.md를 기준으로 Self-Correcting TDD 루프를 시작해줘.

매 반복마다 반드시 다음 순서를 지켜:

1. MEMORY.md를 읽고 목표, 현재 상태, Retry 횟수, 금지사항 확인
2. 해결할 최소 단위 작업 하나 선택
3. Act: 허용된 범위에서 최소 수정
4. Observe: 정의된 Verifier 실행 및 로그 수집
5. Reason: 실패 원인을 분류하고 오류 fingerprint 생성
6. MEMORY.md에 실행 기록과 현재 상태 저장
7. 통과하지 못한 경우 Retry 조건을 확인한 뒤 다음 루프 수행

루프 정책:
- 최대 Retry는 3회
- 동일 오류 fingerprint가 2회 반복되면 즉시 중지
- 한 Retry에서 하나의 원인만 수정
- 테스트 삭제, 테스트 완화, 대규모 리팩토링 금지
- 요구사항, API, 인증·권한, 외부 서비스 판단이 필요하면 HITL_REQUIRED로 중지
- Verifier 전체 통과 시 PASSED로 종료
- 테스트 실행이 불가능하면 BLOCKED로 종료
- 모든 반복 결과를 즉시 MEMORY.md에 기록

가능하면 Claude Code CLI를 독립 Verifier로 사용해.
Sonnet 5 사용 가능 여부를 확인하고 실제 사용한 모델명을 MEMORY.md에 기록해.
비밀키와 토큰 값은 출력하거나 기록하지 마.

GitHub에는 자동으로 push하거나 배포하지 마.
모든 Verifier가 통과하면 변경 파일, 테스트 결과, commit 후보 메시지를 정리하고
상태를 DEPLOY_APPROVAL_REQUIRED로 설정한 뒤 사람의 승인을 요청하며 중지해.

사람이 명시적으로 배포를 승인한 이후에만:
- github_token.txt의 토큰을 화면이나 로그에 노출하지 않고 사용
- 변경사항을 commit 및 push
- GitHub Pages 배포 상태를 확인
- 배포 결과를 MEMORY.md에 기록

완료되거나 중지되면 최종 상태, Retry 횟수, 변경 파일, 테스트 결과,
중지 이유와 [사람 확인 필요] 항목을 보고해줘.
