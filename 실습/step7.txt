현재 프로젝트의 Step 1 분석 결과, AORR.md, MEMORY.md, 기존 저장소 파일을 읽고 Self-Correcting TDD 루프를 정확히 1회만 실행해줘.

이번 단계의 목표는 GitHub Pages에서 실행 가능한 정적 웹사이트의 가장 안전한 기본 구조를 만드는 것이야.

Step 1에서 추천한 첫 번째 루프가 더 안전하고 구체적이라면 해당 루프를 우선해.
그렇지 않으면 다음 작업을 하나의 첫 루프로 수행해:

- 프로젝트 루트의 index.html 생성 또는 기존 파일 최소 수정
- styles.css 생성 또는 기존 스타일 연결
- script.js 생성 또는 기존 스크립트 연결
- 모바일 viewport 설정
- 기본 반응형 내비게이션 생성
- Home, About, Projects 또는 기존 프로필 섹션 연결
- 상단에 Games 탭 또는 Games 링크 영역 생성
- 아직 게임 전체를 완성할 필요는 없지만 Games 영역이 페이지에 존재해야 함

기존 사이트가 있다면 전체를 덮어쓰지 말고 기존 콘텐츠와 구조를 최대한 보존해.

실행 순서:

1. MEMORY.md의 목표, 범위, 가드레일 확인
2. 기존 파일과 Git 상태 확인
3. 변경 전 상태를 Verifier로 확인
4. 이번 루프에서 해결할 최소 작업 하나 확정
5. 필요한 최소 파일만 생성 또는 수정
6. 로컬 정적 서버에서 확인
7. HTML, CSS, JavaScript 연결 확인
8. 모바일 viewport와 기본 반응형 확인
9. 실행 결과를 MEMORY.md에 기록
10. 정확히 한 번의 Act-Observe-Reason 사이클 후 중지

가능하면 Claude Code CLI를 독립 Verifier로 사용해.
Sonnet 5가 실제 사용 가능하면 사용하고, 그렇지 않으면 사용 가능한 Sonnet 모델을 사용한 뒤 실제 모델명을 기록해.

다음 작업은 하지 마:

- 전체 지렁이 게임 구현
- Git push
- GitHub Pages 배포
- 테스트 삭제 또는 완화
- 기존 콘텐츠 대규모 삭제
- 외부 프레임워크로 전환
- 토큰 출력 또는 커밋

최종적으로 다음을 보고해:

- 선택한 첫 번째 루프
- 생성하거나 수정한 파일
- index.html 생성 여부
- 실행한 Verifier
- 테스트 결과
- 현재 상태
- 다음 Step 7에서 수행할 남은 작업

실패하거나 불확실한 항목은 [사람 확인 필요]로 표시해줘.
