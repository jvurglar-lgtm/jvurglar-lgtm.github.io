다음 개발 업무를 루프 엔지니어링 관점에서 분석해줘.

GitHub 저장소 https://github.com/daniel-eai/daniel-eai.github.io.git 를 clone해서 개인 프로페셔널 웹사이트를 만들려고 해. 모바일을 지원하도록 반응형으로 만들고, 상단에 Games 탭을 추가해 키보드와 모바일 터치로 조작할 수 있는 지렁이 게임을 구현하고 싶어. 정적인 웹사이트 만들고 싶어.

[게임 추가 기능:]

각 업무를 작은 개발 루프로 나누고, 입력 자료, Act, Observe, 테스트 기준, 위험도, 사람 확인이 필요한 지점, 첫 루프로 적합한지를 표로 정리해줘. 이후 가장 안전한 첫 번째 루프를 추천해줘.

GitHub 인증이 필요하면 현재 디렉토리의 github_token.txt에 있는 token="" 값을 사용하되, 토큰을 출력하거나 코드에 저장하거나 Git에 커밋하지 마.

아직 코드 수정, push, 배포는 하지 말고 분석 결과만 보여줘.
