# AUTOMATION_READINESS

## 1. 개요
- 대상: `jvurglar-lgtm.github.io` 정적 개인 웹사이트 + `Games` 지렁이 게임
- 현재 기준: Step 5까지 완료, 반응형 정적 진입점과 `Games` 준비 영역은 구현됨
- 기본 원칙: Codex는 최소 수정, Claude Verifier는 동일 기준 검증, 사람은 콘텐츠·디자인·배포 승인 담당

## 3. 지금 자동화하기 가장 좋은 루프
- 추천 루프: **루프 8. 접근성/QA**
- 이유: 현재 구조가 이미 있고, 사람 승인 없이 반복 검증하기 쉽고, 반응형 정적 사이트에서 자동화 효율이 높다.

## 4. 사람이 반드시 승인해야 하는 지점
- 프로필 내용, 직함, 공개 링크
- 디자인 방향, 색상, 타이포그래피
- 게임 규칙, 적 추가 여부, 난이도, 입력 방식
- GitHub Pages 배포 전 공개 승인

## 5. Claude Verifier가 확인해야 할 항목
- 파일 존재와 상대 경로
- HTML 구조와 내부 링크
- CSS 반응형과 모바일 레이아웃
- JavaScript 오류와 게임 상태 전환
- 게임 입력과 재시작 흐름
- 로컬 HTTP 응답과 GitHub Pages 호환성

## 6. Claude 사용 불가 시 Fallback
- `claude_code.exe` 실행이 가능하면 기본 Verifier로 사용
- `ANTHROPIC_API_KEY`가 우선되어 충돌하면 해당 변수를 제거한 뒤 재실행
- 그래도 불가하면 `CODEX_FALLBACK`으로 전환하고, Codex가 변경과 검증을 모두 수행

## 7. 운영 전 필요한 테스트·권한·모니터링
- 테스트: 로컬 열림, 375/768/1440px, 링크 이동, 콘솔 오류, 게임 입력, 재시작
- 권한: GitHub Pages 배포 권한, 저장소 쓰기 권한, 필요 시 토큰 사용
- 모니터링: 배포 직후 404, 자산 경로, 모바일 메뉴, 게임 조작, 콘솔 에러

## 8. 현업 적용 다음 액션 3개
1. Step 6에서 지렁이 게임 핵심 로직의 자동화 경계를 확정한다.
2. 게임 입력 방식은 모바일 조작성 기준으로 사람 승인을 받는다.
3. 배포 전 검증 체크리스트를 고정해 반복 가능한 운영 루프로 만든다.
